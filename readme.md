Описание проекта

Этот проект имитирует реальную задачу Аналитика данных/Инженера данных. Цель проекта - разработать ETL-пайплайн для обработки "сырой" выгрузки транзакций сети городских аптек. Исходные данные содержат ошибки ввода, некорректные типы данных, пропуски и технический мусор.

В ходе работы создается чистый, оптимизированный датасет, готовый для финансовой отчетности или выгрузки в bi-систему.

Технологический стек:
*Python 3+;
*Pandas ;
*NumPy;
*Jupyter Notebook|Colab.


В ноутбуке pharmacy_data_processing_and_preparation.ipynb реализованы следующие шаги:

1. Генерация синтетических данных: Создание реалистичного датасета (4000 строк) с намеренно внесенными аномалиями (грязные данные).

2. Очистка данных:
	Обработка пропусков (NaN) на основе бизнес-логики (удаление критических, заполнение 	заглушками некритических);
	Удаление технического "шума" и нерелевантных столбцов;

3. Фичи проектирования:
	Парсинг и объединение даты/времени из строковых форматов;
	Извлечение полезных признаков (месяц, день недели);

4.Оптимизация памяти:
	Даункаст числовых типов (float64 -> float32);
	Преобразование строковых объектов в category (результат: 
	снижение потребления памяти в 	несколько раз).

5. Фильтрация и Экспорт:
	Подготовка финального среза данных  для отчета.

Результаты оптимизации.
Благодаря изменению типов данных (dtypes optimization), объем потребляемой памяти датафреймом был уменьшен:

До обработки: ~2.5 MB (условно);
После обработки: ~0.4 MB;
Сжатие: в ~6 раз.
